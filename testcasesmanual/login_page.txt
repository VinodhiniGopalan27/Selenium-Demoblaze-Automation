Functional Test Scenarios for Login Page:
1.	Verify that the user can login by entering valid credentials and clicking on the ‘Login’ button.
2.	Verify whether the user is not able to login with an invalid username and password.
3.	Check that the validation message is displayed if the user exceeds the character limit of the username and password fields.
4.	Check if the password is in masked form when typed in the password field.
5.	Verify that the validation message saying “Please fill out the username and password” gets displayed if the user leaves the username or password field blank.
6.	Verify that closing the browser should not log out an authenticated user.
7.	Verify if there is a checkbox with the label “remember password” on the login page.
8.	Verify that there is a limit on the total number of unsuccessful login attempts.
9.	Verify the login session timeout duration.
10.	Verify that once logged in, clicking the back button doesn’t log out the user.
11.	Verify that the reset button functionality is on the login page. Clicking on it should clear the textbox’s content.
